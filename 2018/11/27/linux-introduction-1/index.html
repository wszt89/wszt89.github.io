常用命令

线上出问题了第一反应是看日志，常用的几个命令有：

  • vim、vi：准确的说是编辑器，在打开大的文件会占用很大内存（把文件全部的内容载入到内存，有插件的话消耗更大），线上操作有风险；
  • less：支持前后翻页、支持搜索，比较适合用来看日志；
  • more：类似less，但不支持后退；
  • cat：显示文件所有的内容，也可以用来做文件合并；
  • head：显示文件头几行；
  • tail：显示文件最后几行，而且可以实时看文件变化，用的频率也非常高；

less常用的指令有：

  • 回车：向下移动一行；
  • y：向上移动一行；
  • 空格：向下滚动一屏；
  • b：向上滚动一屏；
  • d：向下滚动半屏；
  • u：向上滚动半屏；
  • p：指定百分比跳转；
  • g：跳到第一行；
  • G：跳到最后一行；
  • /pattern：查找短语，支持正则；
  • v：调用vi编辑器；
  • q：退出；

tail常用的指令有：

  • -f：在文件内容增加后自动显示；
  • -10：展示行数；

比较好玩的是可以实时查看多个文件：tail -f ./*.log。

从文件中找需要的内容，除了使用/pattern外，还可以使用grep命令：

  • -A：显示前面的行数；
  • -b：显示后面的行数；
  • -C：显示前后的行数；
  • -v：反转查询（不匹配）；
  • -E：正则表达式匹配模式；
  • -c：统计匹配到的数量；
  • -n：展示行号；
  • -H：展示文件名；
  • -i：匹配是忽略大小写；
  • -r：递归处理；
  • -e：指定多个查找内容；

性能

查看CPU的使用情况用top，展示信息的含义是：

  • load average：最近1/5/15分钟系统的平均负荷，小于1表示系统有一定的空闲，大于1表示任务需要排队等待处理；
  • Tasks
    • total：进程总数；
    • running：运行中的进程数；
    • sleeping：等待状态的进程数；
    • stop：被停止的进程数；
    • zombie：被复原的进程数；
  • %Cpus
    • us(user CPU time)：用户空间占用CPU百分比；
    • sy(system CPU time)：内核空间占用CPU百分比；
    • ni(nice CPU time)：用户进程空间内，改变过优先级的进程占用CPU百分比；
    • id(idle)：空闲CPU百分比；
    • wa(iowait)：等待输入输出的CPU时间百分比（网络和磁盘等是不是有问题）；
    • hi(hardware irq)：硬件中断；
    • si(software irq)：软件中断；
    • st(steal time)：实时占比；
  • KiB Mem
    • total：物理内存总量；
    • free：空闲大小；
    • used：使用中的大小；
    • buff/cache：缓存的内存量；
  • KiB Swap
    • total：交换区总量；
    • free：空闲大小；
    • used：使用中的大小；
    • avail Mem：可用于进程下一次分配的物理内存数量；
  • PR：优先级，越小优先级越高；
  • VIRT：虚拟内存；
  • RES：物理内存（正在使用、没有交换的内存）；
  • SHR：共享内存（和其他进程共享的内存）；
  • S：进程的状态
    • S：可中断的休眠（等待Socket连接等，在相应的事件发生时会被唤醒）；
    • D：不可中断的休眠（保护内核的某些处理流程不被打断）；
    • R：可执行；
    • T：暂停或跟踪；
    • Z：僵尸进程，资源已经被回收，只剩元数据；
  • %CPU：占用CPU的比例；
  • %MEM：占用的物理内存和总内存的比例；
  • TIME+：进程启动后占用的CPU时间的总和；
  • COMMAND：进程启动命令名称；

修改展示方式的指令有：

  • N：按照PID排序；
  • P：按照CPU占用率排序；
  • M：按照内存占用率排序；
