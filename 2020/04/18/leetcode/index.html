最长上升子序列

思路：

动态规划

状态转移方程为dp[i] = max{dp[j] + 1}，其中a[j] > a[i]，复杂度为O(N^2)​。

可以使用树状数组把状态转移方程的复杂度降低到O(logN)，总体复杂度降低到O(N*logN)。

贪心+二分

相同的长度，最后一个数字越小越好。用low[i]表示长度为i的最后一个数的最小值，于是一定有low[i+1] > low[i]。

对于a，找到low最大的小于a的数i，更新low[i+1] = a。

题目：

  • https://leetcode-cn.com/problems/circus-tower-lcci/

接雨水

接雨水：https://leetcode-cn.com/problems/trapping-rain-water/

从左往右看，如果右边柱子比左边柱子高，那么左边柱子对后续的柱子放水没有任何影响。可以考虑用单调递减栈来保存柱子。

遍历柱子，如果当前柱子大于栈顶元素，在出栈前计算与栈顶柱子间能放的水量。如果柱子要入占了，需要考虑当前柱子很栈顶柱子之间的水量。

最大矩形

柱状图中的最大矩形：https://leetcode-cn.com/problems/largest-rectangle-in-histogram/

可以用分治$O(NlogN)$来解决：找到最小的的，分别计算左、右及全部的来选择最大值。

**

可以用$O(N^2)$的复杂度来遍历解决：

  • 遍历每个柱子；
  • 找到每个柱子左边的边界和右边的边界；

**

如有右边的柱子大于左边的柱子，那么右边柱子的左边界就是左边的柱子。可以用单调递增栈来保存柱子的左边界，如果当前柱子大于栈顶元素，也就是找到栈顶元素的右边界：

  • 右边界：当前位置；
  • 左边界：栈里面前一个元素的位置；
  • 高度：栈顶值；

Tips：先把-1压栈，当做哨兵。最后把-1压栈，用来把栈清空掉。

下一个序列

https://leetcode-cn.com/problems/next-permutation/

解法：

  • 从后往前，找到第一个开始下降的（如果不存在下降的，那么说明已经是最大的序列）；
  • 左边的最后一个值为$x$；
  • 找到右边最后一个$y>x$（这样的$y$是最小的），将$x,y$进行交换，这样对于字典序的增加是最小的，再将右边的序列进行翻转；

栈

最小字典序子串：https://leetcode-cn.com/problems/remove-duplicate-letters/

因为有序，可以考虑用单调递增栈来保存结果。

遍历字符串，比较当前字符和栈顶字符：

  • 大于：入栈，保持递增；
  • 小于(循环)：判断栈顶元素是不是最后一次出现：
    • 是：跳出循环，这样会破坏了栈的递增，前面的字符顺序已经不能修改，所以在每次处理前，如果字符已经在栈里面就不用管了；
    • 否：出栈，后面还有机会加回来，但是这次出栈可以保持递增；

最大频率栈（先数量再次序）：https://leetcode-cn.com/problems/maximum-frequency-stack/

比较麻烦的是如何处理最后入栈的值，最后出现的是在栈里面最后面的。

可以为每个数量维护一个数字的栈就行。

队列的最大值：https://leetcode-cn.com/problems/dui-lie-de-zui-da-zhi-lcof/

PUSH和POP天然就是$O(1)$，比较麻烦的是MAX。

在一个大的值进PUSH后，前面PUSH的小的值就没有影响了：单调递减维护在队列中的数据，每次查到的最大值就是头上的。

如果在POP的时候是最大值，最大值数量减一，最大值的数量在两个队列里面永远都是一致的。

批量操作的栈：https://leetcode-cn.com/problems/design-a-stack-with-increment-operation/

最直观的是在INCR的时候占用的时候使用$O(N)$的复杂度修改。

**

如果在INCR的时候记录要加的位置和数量，在POP的时候把增加的数量从高位上推到低的位置上，每个操作的复杂度为$O(1)$。

索引处的解码字符串：https://leetcode-cn.com/problems/decoded-string-at-index/

根据题目升生成的序列为：

a = 1, 2, 3, 4, 8[=4 x 2], 9, 10, 11, 12, 36[=12 x 3]

b = l, e, e, t, 2, c, o, d, e, 3​

依次来计算a[i]，当a[i] > k时，说明字符一定在前面的字符出现过了，倒着开始遍历处理。编码技巧：一步一步来看，把字符串可以转换成一个数组（试试转换）。

132模式：https://leetcode-cn.com/problems/132-pattern/

需要$i<j<k$且$a_i<a_k<a_j$，从$a_k$的角度来考虑：

  • 找到左边第一个小于$a_k$的坐标$x$；
  • 找到左边末一个大于$a_k$的坐标$y$；

如果存在$x<y$，那么就存在，否在不存在。分别用递增和递减栈，貌似需要$O(nlogn)$的时间。

**

从$a_{j}$的角度来考虑：

维护$min[i]=min(a[0],…,a[i])$，如果$a_j<=min[j]$，说明不存在$a_i < a_j$。

如果$min[j] >= a_k$则一定有$min[j-1] >= a_k$，于是不存在$ai_i<a_k$。

比较$a_j,a_k$：

  • $a_j > a_k$：找到答案；
  • $a_j < a_k$：压入栈中，因为$a_k$为了满足条件压入栈中，栈顶永远是后面满足条件最小的$a_k$；

丑数：https://leetcode-cn.com/problems/super-ugly-number/

为每个乘数维护一个指针，指向该乘数考虑的最小的位置。然后根据乘完的结果维护一个小根堆。

  • 从小根堆中取出最小的节点；
  • 乘积就是需要第$n$个结果，然后沿着结果列表移动指针；

Tips：需要用过集合判重。

**

朴素的思路是：取出最小的一个，然后分别乘以各个数字丢进小根堆。问题在于：如果乘完的结果很大，其实暂时没必要考虑。

排和列都有序的K个数：https://leetcode-cn.com/problems/kth-smallest-element-in-a-sorted-matrix/

直观的思路是用堆。

**

用二分来找数，Left=左上角的最小值，Right=右下角的最大值。计算出Mid后在有序的矩阵中统计数量。

带随机指针的链表复制：https://leetcode-cn.com/problems/copy-list-with-random-pointer/

通用的数据结构复制套路，用一个MAP记录访问过的节点，然后递归处理指针：

**

更巧妙的思路、更合理：

  • 先沿着Next复制，并嵌入到原来的链表里面，这样和原有的链表也建立的一对一的关系；
  • 然后依次处理随机指针；

如下图：

  • 栈
    • 维护有序的结果；
      • 有序值
      • 最值（辅助栈）
    • 寻找已出现的最新的边界值；
      • 对于单调递增栈的边界：$a_{new}>=a_{top}$时入栈，$a_{new}<a_{top}$时出栈；
        • 入栈：$a_{top}$是前面最后一个小于$a_{new}$的值；
        • 栈中：
          • $a_{i-1}$是前面最后一个小于$a_i$的值（累积的入栈顺序，区别在于栈中只保存了部分）；
          • $a_{i}$是后面最小的大于$a_{i-1}$的值；
        • 出栈：$a_{new}$是后面第一个小于$a_{top}$的值；
      • 对于单调递减栈的边界：$a_{new}<=a_{top}$时入栈，$a_{new}>a_{top}$时出栈；
        • 入栈：$a_{top}$是前面最后一个大于$a_{new}$的值；
        • 栈中：
          • $a_{i-1}$是前面最后一个大于$a_i$的值（累积的入栈顺序，区别在于栈中只保存了部分）；
          • $a_{i}$是后面最大的小于$a_{i-1}$的值；
        • 出栈：$a_{new}$是后面第一个大于$a_{top}$的值；
  • 队列
    • 维护递减的滑动窗口，用来在$O(1)$的时间获取最大值；
  • 堆
    • 小根堆：最大的第K个数（或者是一个结构体：比如丑数）；
    • 大根堆：最小的第K个数；
  • 二分
    • 根据最大值和最小值进行二分
    • 根据范围进行二分
  • 贪心
  • 链表
    • 环
      • 快慢指针
      • 环的大小：LEN；
      • 先后指针找环出现的位置：一个指针先走LEN，然后两个指针一起走，相遇时就是交叉点；
    • 链表翻转：栈、递归、多指针；
    • 转换为有头链表
    • 复制：递归、用MAP保存访问过的节点；
  • 位操作
  • 动态规划
    • 最大和子序列
    • 通用博弈归纳：下一步到的都是必败的，那么当前这一步就是必胜的；
  • 其他
    • 超过一半的数：相同加一，不同减一，等于0时换人，保证不会错过候选人；
