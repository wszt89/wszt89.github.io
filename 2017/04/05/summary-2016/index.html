总结：2016

发表于 2017-04-05   |   分类于 总结   |  

在菜鸟的第三个年头，有很多的不如意和更不如意，不管结果怎么样，都值得花一段时间认真地思考和总结。

项目总结

之前一直在做数据，从16年5月开始搞业务，不得不说菜鸟的业务还是重，而仓库里面的业务更重。

退供

这是第一个深入接触的业务，流程比较简单：

  • 接单
  • 下架
  • 质检
  • 出库

但是等系统的实现捋了一遍之后，简直丧心病狂，没有出故障真是奇迹。还没消停几天就问题来了：

  • 单据状态出错
  • 单据数据出错

这些都忍了，订正一下就完了。结果又来了一个：

有个商品数几万的单子，被下架了两次，然后很多下架单都被作业了，问题是系统上看起来多出库了。

于是整整一个礼拜都在想办法从所有的明细中找到出错的，功夫不负有心人，终于找到了，而根本的原因也是让人哭笑不得：

整个系统的事务都是不生效的。

集货仓

集货是一个很有意思的项目:

集货仓作为一个中心仓，为前置仓供货。

如果真能很好地执行，那么前置仓可能就不需要囤货了。想想很激动，但是项目过程比较心累：

  • “行业换老板了”
  • “新项目起来，要搞新项目去了”
  • “需要资源随便提！”“XXX可以把那个模块实现下吗？”“这个不行！”

而且一些系统问题实在太多，甚至个别开发一周的时间只能投入两、三个小时，跟老板要资源也是打哈哈。等到项目上线并且反馈还不错的时候：

那些不太积极的人，又积极起来了。
