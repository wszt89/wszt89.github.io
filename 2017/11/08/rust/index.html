Rust

发表于 2017-11-08   |   分类于 编程   |     |   阅读次数

现代的编程语言，会有各种“奇技淫巧”来接管内存的释放，较大的一支是通过垃圾回收，然而：

大部分的垃圾回收会导致或多或少的卡顿。

在一些场景中是无法忍受的，Rust似乎是C/C++之外一种不错的选择。

起步

一键安装：

将在你的电脑上安装：

  • cargo：管理依赖包，有点像Maven；
  • rustc：编译器；
  • rustup：Rust版本管理工具，语言本身迭代非常快，工具还是非常有必要的，另外用来做交叉编译也很简单；

优势

用惯了自带GC的语言，再也不想手动管内存了。而RUST则既不用管内存释放、也没有GC，优势在于：

  • Cover无法忍受GC过程中Stop The World导致停顿的场景（点亮技能树）；
  • 方便地交叉编译；
  • 安全的内存管理；
  • 跟随语言一起成长的机会；
  • 从语言设计的理念中，学习一些编程的最佳实践；

总的来说，花点时间学习RUST还是非常值得的。

作用域和生命周期

必须要弄得非常清楚的几个概念：

  • 所有权
    • 每个值都有且仅有一个所有者；
    • 当所有者离开作用域，值会被销毁；
  • 移动：let mut x = y
    • 资源的所有者发生变换，老的所有者不能使用，避免悬垂指针；
    • 能够改变可变性；
  • 借用：let x = &mut y、let ref mut x = y
    • 访问值，但不会变为它的所有者；
    • 被借用后，所有者不能释放、修改值；
    • 允许有多个借用；
    • 能够改变可变性，限制：不允许借用不可变为可变；
    • 不可变借用可以冻结可变值；
    • 可多次不可变借用、期间不允许可变借用；
    • 同一时间只允许一个可变借用；

而Rust的精髓就在于：

通过所有权、借用，在生命周期的框架下安全地使用内存。

总结

Rust不想引入GC的前提下解决内存安全的问题，思路是：

让逻辑发生在可控范围内，危险的简便写法宁可不要。

增加了不小的学习成本和使用成本，但是，对做大型的项目来说，收益还是非常吸引人的。
